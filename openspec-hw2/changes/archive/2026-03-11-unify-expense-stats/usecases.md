# 用例：统一记账统计接口

## UC1：查询统计数据

**主角**：客户端（Web / Android / 鸿蒙）

**前置条件**：用户已登录

### 主场景

1. 客户端发送 `GET /api/expenses/stats?period=month&date=2026-03-11`
2. 后端根据 period 计算日期范围（day/week/month）
3. 后端查询该范围内用户的所有记账条目
4. 后端聚合计算：
   - 总金额、条目数
   - 按原始标签聚合（tag_totals）
   - 按分类映射聚合（category_totals），含百分比
   - 按日期聚合（daily），空日期填 0
   - 查询上一周期数据，计算环比（comparison）
5. 后端返回统一的 JSON 响应
6. 客户端按需取字段渲染（标签筛选、分类饼图、趋势柱状图、环比指标）

### 扩展

- **1a. 未传 date 参数**：默认使用今天日期
- **1b. 未传 period 参数**：默认使用 "month"
- **3a. 该范围无数据**：返回 total_amount=0，空数组，comparison 中 prev_total 也查询返回
- **4a. 标签无法映射到分类**：归入"其他"分类

## UC2：废弃旧接口的平滑过渡

**主角**：客户端

**前置条件**：旧接口 `/summary` 和 `/analytics` 仍在使用

### 主场景

1. 后端保留 `/summary` 和 `/analytics` 接口，行为不变
2. 在 API 文档中标注 deprecated，指向新接口
3. 各端在适配新接口后，逐步停止调用旧接口
4. 旧接口在所有端切换完成前不删除

### 扩展

- **3a. 某端暂时无法适配**：继续使用旧接口，不受影响

## UC3：LLM 工具调用统计

**主角**：用户（通过对话）

**前置条件**：用户已登录，有记账数据

### 主场景

1. 用户对二狗说"这个月花了多少钱"
2. 二狗调用 `expense_summary` 工具，工具内部请求 `/api/expenses/stats?period=month`
3. 工具从响应中提取 total_amount、category_totals、comparison
4. 工具格式化为自然语言返回给二狗
5. 二狗回复用户，包含总额、分类明细、环比变化

### 扩展

- **2a. 用户指定标签**：工具加 `tag` 参数筛选
- **2b. 用户问上周/今天**：工具调整 period 参数
